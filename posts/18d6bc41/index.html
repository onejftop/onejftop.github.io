MySQL 数据库基本操作

1970-01-01- 2021-03-01
683- 2m
MySQL-MySQL数据库

数据库操作

创建数据库

# “库选项”用于设置此数据库的相关特性，如字符集CHARSET，校对集COLLATE。
CREATE DATABASE 数据库名称 [库选项];
# 该命令表示当指定的数据库不存在时，执行此操作，否则忽略此操作。
CREATE DATABASE IF NOT EXISTS 数据库名称;

查看数据库

查看所有数据库

SHOW DATABASES;

查看指定数据库的创建信息

SHOW CREATE DATABASE 数据库名称;

选择数据库

USE 数据库名称;
# 登录时，可以直接选择要操作的数据库。
mysql -u用户名 -p密码 数据库名

删除数据库

# IF EXISTS：若要删除的数据库存在，则执行此操作，否则不执行此操作
DROP DATABASE [IF EXISTS] 数据库名称;

数据表操作

创建数据表

CREATE [TEMPORARY] TABLE [IF NOT EXISTS] 表名
(字段名 字段类型 [字段属性]...) [表选项]

TEMPORARY：表示临时表。

表选项：用于设置表的相关特性，如存储引擎（ENGINE）、字符集（CHARSET）和校对集（COLLATE）。

# 使用“数据库.表名”的方式可以在任何数据库下访问其他数据库中的表
CREATE TABLE 数据库名称.表名 (此处省略字段的定义);

查看数据库表

查看数据表

SHOW TABLES [LIKE 匹配模式];

上述语法中，若不添加可选项“LIKE 匹配模式”则表示查看当前数据库中的所有数据表;若添加则按照“匹配模式”查看数据表。其中，匹配模式符有两种，分别为 “%” 和 “_”。前者表示匹配一个或多个字符，代表任意长度的字符串，长度可以为0,后者仅可以匹配一个字符。

注意：LIKE后的匹配模式必须使用单引号或双引号包裹。

查看数据表的相关信息

SHOW TABLE STATUS [FROM 数据库名] [LIKE 匹配模式];

修改数据表

修改数据表名称

# 方式一
ALERT TABLE 旧表名 RENAME [TO|AS] 新表名;
# 方式二
RENAME TABLE 旧表名1 TO 新表名1 [,旧表名2 TO 新表名2]...

修改表选项

ALERT TABLE 表名 表选项 [=] 值;

查看表结构

查看表的字段信息

# 查看所有字段的信息
{ DESCRIBE | DESC } 数据表名;
# 查看指定字段的信息
{ DESCRIBE | DESC } 数据表名 字段名;

查看数据表的创建语句

SHOW CREATE TABLE 表名;

查看数据表结构

# 方式一
SHOW [FULL] COLUMNS FROM 数据表名 [FROM 数据库名];
#  方式二
SHOW [FULL] COLUMNS FROM 数据库名.数据表名;

上述语法中，可选项 FULL 表示显示详细内容，在不添加的情况下查询结果与 DESC 的结果相同。
